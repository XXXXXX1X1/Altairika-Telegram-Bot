# 02 — Пользовательские сценарии

## Нотация

```
[Экран / сообщение]  →  (Действие пользователя)  →  [Следующий экран]
```

Знак `✎` = сообщение редактируется на месте (не отправляется новое).
Знак `+` = отправляется новое сообщение.

---

## Сценарий 1 — Знакомство с каталогом и запись на сеанс

**Персона:** A — потенциальный клиент.
**Цель:** найти подходящую позицию и записаться.

```
[Старт: /start или первое сообщение]
    Приветственное сообщение + главное меню (inline-кнопки)
    ┌──────────────────────────┐
    │ 📋 Каталог               │
    │ 🏢 Франшиза              │
    │ ❓ Задать вопрос         │
    │ 📞 Связаться             │
    └──────────────────────────┘

(нажимает «Каталог»)  →  ✎ [Экран каталога]
    Краткое описание + фильтры (inline-кнопки по категориям)
    [Все] [Категория 1] [Категория 2] ... — сформированы по данным

(выбирает категорию)  →  ✎ [Список карточек в категории]
    Пагинация: «← 1/5 →», краткие названия с кнопкой открыть

(нажимает на карточку)  →  ✎ или + [Карточка]
    Если есть изображение → + новое сообщение с фото и caption
    Если нет изображения → ✎ то же сообщение с текстом
    
    Содержание карточки:
    ─────────────────────
    [Изображение / постер — если есть]
    
    Название
    Краткое описание (2–4 строки)
    Подробности: длительность, формат, категория
    Цена (если есть в данных)
    ─────────────────────
    [Записаться]  [← Назад к списку]
    [Похожие]     [Задать вопрос об этом]

(нажимает «Записаться»)  →  + [Форма записи]
    «Оставьте ваше имя и номер телефона, и мы свяжемся с вами.»
    
    Шаг 1: «Как вас зовут?» (текстовый ввод)
    Шаг 2: «Ваш номер телефона?» (кнопка "Поделиться контактом" + ввод вручную)
    Шаг 3: Подтверждение — «Записать [Имя] на [Название], телефон [X]?»
            [Да, всё верно] [Изменить]
    
    После подтверждения → + [Финальное сообщение]
    «Спасибо! Мы получили вашу заявку. Ожидайте звонка в рабочее время.»
    [← Вернуться в каталог] [Главное меню]
```

## Сценарий 2 — Вопрос о франшизе

**Персона:** B — потенциальный франчайзи.
**Цель:** получить информацию и оставить контакт.

```
[Главное меню]

(нажимает «Франшиза»)  →  ✎ [Раздел франшизы]
    Краткий питч (3–5 строк о предложении)
    ┌─────────────────────────────┐
    │ 💰 Условия и инвестиции     │
    │ 🤝 Поддержка франчайзера    │
    │ 📊 Рынок и конкуренты       │
    │ 📋 Частые вопросы           │
    │ 📞 Оставить заявку          │
    └─────────────────────────────┘

(нажимает «Условия и инвестиции»)  →  ✎ [Текст условий]
    Структурированный текст по блокам
    [← Назад] [Оставить заявку]

(нажимает «Рынок и конкуренты»)  →  ✎ [Сравнение]
    Таблица или список: Altairika vs N конкурентов
    По ключевым параметрам (сформировано из открытых данных)
    [← Назад] [Задать вопрос]

(нажимает «Оставить заявку»)  →  + [Форма заявки на франшизу]
    Шаг 1: Имя
    Шаг 2: Телефон (кнопка "Поделиться" или ввод)
    Шаг 3: Город / регион (текстовый ввод)
    Шаг 4: Подтверждение
    
    После → «Заявка отправлена. Наш менеджер свяжется с вами в течение 1 рабочего дня.»
    [← Главное меню]
```

> [Решение] Данные о франшизе (инвестиции, паушальный взнос, роялти) не фиксируются в боте как константы — они подтягиваются из заранее подготовленного структурированного текста, который можно обновлять без деплоя.

> [Требует данных] Конкретные условия франшизы (суммы, сроки, формат поддержки) получаются от заказчика или с сайта.

## Сценарий 3 — FAQ

**Персона:** A / B / C — любая.
**Цель:** получить ответ на типичный вопрос без навигации по каталогу.

```
[Главное меню → «Частые вопросы»]

→ ✎ [Список тем FAQ]
    ┌──────────────────────────────┐
    │  О сеансах и записи          │
    │  О франшизе                  │
    │  Цены и оплата               │
    │  Другой вопрос               │
    └──────────────────────────────┘

(выбирает тему)  →  ✎ [Список вопросов в теме]
    1. Вопрос A
    2. Вопрос B
    3. Вопрос C
    [← Назад к темам]

(выбирает вопрос)  →  ✎ [Ответ]
    Текст ответа
    [← Назад к вопросам]  [📞 Задать свой вопрос]

(нажимает «Задать свой вопрос»)
    → + «Напишите ваш вопрос — мы ответим в рабочее время.»
      Пользователь пишет текст → сохраняется как обращение → оператор получает уведомление
      → «Спасибо! Мы ответим вам в ближайшее время.»
```

> [Требует данных] Темы и вопросы FAQ формируются совместно с заказчиком. В коде ничего не зашивается — всё хранится в БД.

> [Решение] Свободный текстовый ввод вне формы → бот предлагает перейти в FAQ или написать вопрос оператору. Без разбора намерений.

## Сценарий 4 — Пользователь хочет только оставить контакт

**Персона:** C — уже готов к контакту, не хочет изучать каталог.

```
[Главное меню → «Связаться»] или [/contact]

+ [Короткая форма]
    «Оставьте контакт — мы перезвоним и ответим на все вопросы.»
    Шаг 1: Имя
    Шаг 2: Телефон
    Подтверждение → «Готово. Ждите звонка.»
```

## Сценарий 5 — Сравнение с конкурентами

**Персона:** C — исследователь.
**Цель:** убедиться, что Altairika — лучший выбор.

```
[Франшиза → Рынок и конкуренты]
или
[Свободный вопрос: «Чем вы лучше X?»]

→ [Экран сравнения]
    Таблица: Altairika | Конкурент 1 | Конкурент 2 | ...
    Параметры сравнения: определяются по открытым данным
    
    Под таблицей: краткий вывод в пользу Altairika
    [Записаться] [Задать вопрос] [← Назад]
```

> [Гипотеза] Список конкурентов и параметры сравнения формируются при сборе данных (парсинг, исследование). Не фиксируются жёстко в документации — обновляются при изменении рынка.

> [Решение] Сравнение доступно как из раздела франшизы, так и в ответ на прямой вопрос. Отдельного пункта в главном меню нет — не нужно выносить его на первый уровень.

## Краевые сценарии

### Пользователь прерывает форму

```
Если пользователь нажимает кнопку меню или пишет что-то постороннее в середине формы:
→ «Вы начали оформление заявки. Продолжить или выйти?»
  [Продолжить] [Выйти]
```

### Пользователь возвращается после паузы

```
Если пользователь пишет /start повторно:
→ Главное меню (состояние сессии сбрасывается, форма не сохраняется)
```

> [Решение] Частичные данные форм не сохраняются между сессиями — упрощает реализацию и не создаёт путаницы. На старте это оправданно.

### Бот не знает ответа

```
→ «Я не нашёл ответ на этот вопрос. Хотите, чтобы наш менеджер связался с вами?»
  [Да, перезвоните мне] [Нет, спасибо]
```
