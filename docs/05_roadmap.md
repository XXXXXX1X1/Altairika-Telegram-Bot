## Фаза 8 — AI-ассистент

> Реализовано в апреле 2026. Затрачено ~5 часов разработки: реализация LLM — 3 ч., тестирование — 2 ч.

### Что реализовано

**Точка входа**
- `freetext_router` теперь запускает AI-pipeline вместо заглушки
- Если `OPENROUTER_API_KEY` не задан — старая заглушка с кнопками FAQ/оператор
- Навигационные команды («открой каталог», «в меню» и т.д.) обрабатываются без LLM

**Intent router (эвристики)**
- `ai_router.py` — скоринг по ключевым словам, 9 интентов
- `ai_decision.py` — LLM-маршрутизатор (первый вызов модели), confidence-порог 0.45
- `ai_branch.py` — логика продолжения ветки диалога без LLM

**Подбор фильмов**
- `ai_movie_params.py` — AI-извлечение параметров (тема, класс, возраст, длительность) с fallback на regex
- `ai_catalog.py` — точный поиск по названию, семантический поиск, AI-ранжирование кандидатов
- `ai_movie.py` — FSM-флоу с листанием карточек, «другая тема», «уточнить», «назад»

**Сессионная память**
- Таблица `ai_sessions` в PostgreSQL (миграция `0003_ai_sessions.py`)
- Хранит: active_intent, параметры подбора, последние 8 сообщений истории
- TTL: 30 минут (настраивается через `AI_SESSION_TTL_MINUTES`)

**Генерация ответа**
- `ai_answer.py` — полный pipeline: память → контекст → системный промпт → LLM → сохранение
- `ai_context.py` — минимальный контекст по intent (не весь каталог, не весь FAQ)
- Базовые правила ответа + инструкция по intent

**Поддерживаемые сценарии**
- Подбор фильмов по теме, классу, возрасту, длительности
- Вопросы о конкретном фильме (точный поиск + похожие варианты)
- О компании, FAQ, франшиза, сравнение с конкурентами
- Заявки: бот не тратит LLM-токены, сразу предлагает форму

## Фаза 7 — Админский раздел

### Реализованные возможности

**Доступ**
- Команда `/admin`
- Проверка по `ADMIN_TELEGRAM_ID` на каждом handler
- Для неадмина: «Доступ запрещён»

**Заявки**
- Список с сортировкой: new → in_progress → done (внутри — новые сверху)
- Пагинация 10/страница
- Карточка с полными данными
- Смена статуса: «В работу» / «Закрыть»

**Вопросы**
- Список (неотвеченные сначала, потом отвеченные)
- Пагинация 10/страница
- Карточка с полным текстом и историей ответа
- Ответ пользователю через FSM (AdminStates.waiting_reply)
- «Отметить отвеченным» без отправки ответа
- Если сообщение пользователю не доставлено — статус не меняется

**Статистика**
- Периоды: Сегодня / 7 дней / 30 дней / Всё время
- Метрики: пользователи, заявки (по типам), вопросы, аналитические события

**Аналитика**
- Таблица `analytics_events`
- Логируются: open_main_menu, open_catalog, open_catalog_item, start_lead_form, submit_lead, ask_question

## Следующие приоритеты

| Задача | Сложность | Ценность |
|--------|----------|---------|
| Webhook-режим вместо polling | Средняя | Снижение latency |
| Redis для FSM и кеша | Средняя | Масштабируемость |
| Автоматический парсер по расписанию | Средняя | Актуальность каталога |
| Аналитика AI-запросов (события intent_detected, ai_answer_sent) | Низкая | Мониторинг качества |
| Инвалидация кеша базы знаний без перезапуска | Низкая | Удобство редактирования |
| pgvector + semantic search | Высокая | Качество подбора |
| Шифрование персональных данных | Средняя | Compliance |
| Экспорт заявок в CSV | Низкая | Операционное удобство |

## Ограничения текущей реализации

- Один администратор (ADMIN_TELEGRAM_ID)
- MemoryStorage для FSM (сбрасывается при перезапуске, AI-сессии сохраняются в PG)
- Нет Redis (кеширование и FSM для multi-instance)
- Нет webhook-режима
- База знаний компании кешируется в памяти — редактирование требует перезапуска
- Нет vector search — подбор через keyword/token matching + LLM-ранжирование
