# 11. AI-сервис — служебный документ

> **[СЛУЖЕБНЫЙ]** Этот документ описывает внутреннюю структуру AI-слоя: конфигурацию модели, системные промпты, формат контекста и сессий. Предназначен для разработчиков и технических ревьюеров.

---

## Конфигурация модели

**Provider:** [OpenRouter](https://openrouter.ai) — OpenAI-совместимый API  
**SDK:** `openai` (AsyncOpenAI, base_url=`https://openrouter.ai/api/v1`)  
**Таймаут:** 20 секунд  
**Конфиг (`.env`):**

```env
OPENROUTER_API_KEY=sk-or-...
AI_MODEL=google/gemini-2.0-flash-001
AI_MAX_TOKENS=600
AI_SESSION_TTL_MINUTES=30
```

Модель меняется без перекомпиляции — достаточно поменять `AI_MODEL` в `.env`. Любая OpenRouter-модель совместима с текущим клиентом.

## Системные промпты

### 1. Базовый промпт ответа (`ai_answer.py`)

```
Ты — помощник компании Альтаирика в Telegram-боте.

Правила:
- Отвечай только по данным из переданного контекста
- Не выдумывай факты, цены, условия, названия фильмов
- Если данных нет — скажи честно и предложи написать оператору
- Отвечай кратко: 3-6 предложений, без воды
- Тон: дружелюбный, живой, не официальный
- Язык: только русский
- Если вопрос не про Альтаирику — мягко верни к теме компании
- Не называй конкретные цены без данных в контексте
- Сообщай факты напрямую, не ссылаясь на источник

--- ДАННЫЕ АЛЬТАИРИКИ ---
{context}
```

К базовому промпту добавляется инструкция по intent (см. таблицу ниже).

### 2. Инструкции по intent (`_INTENT_INSTRUCTIONS`)

| Intent | Инструкция |
|--------|-----------|
| `general_chat` | Поздоровайся, предложи выбрать направление. Не перечисляй фильмы. |
| `movie_selection` | Выбери 2-3 подходящих, объясни почему. Если мало параметров — один уточняющий вопрос. |
| `movie_details` | Только по найденному фильму. Если точного нет — предложи похожие. |
| `franchise_info` | По данным franchise_content. В конце CTA на заявку. |
| `competitor_compare` | Из таблицы сравнения. Не добавляй цифры, которых нет. Без markdown-списков со звёздочками. |
| `lead_booking` | Подтверди намерение, кнопка ниже. |
| `lead_franchise` | Ответь кратко, кнопка ниже. |
| `faq_answer` | Конкретно по FAQ и базе знаний. |
| `company_info` | По базе знаний. Если спрашивают контакты/реквизиты — только если есть в контексте. |

### 3. Промпт LLM-маршрутизатора (`ai_decision.py`)

Отдельный системный промпт для первого вызова (routing). Описывает:
- допустимые intent (9 штук)
- допустимые action (6 штук)
- правила переключения веток
- особые случаи (franchise_info vs lead_franchise, movie_selection vs movie_details)

Возвращает JSON, `max_tokens=220`.

### 4. Промпт извлечения параметров (`ai_movie_params.py`)

Мини-промпт для извлечения параметров подбора фильма из запроса. Возвращает JSON, `max_tokens=180`.

```json
{
  "theme": null,
  "grade": null,
  "age": null,
  "audience": null,
  "duration": null,
  "needs_clarification": false,
  "clarification_reason": ""
}
```

### 5. Промпт AI-ранжирования фильмов (`ai_catalog.py`)

Используется только при семантическом поиске (когда тема не попала в keyword-маппинг). Получает список кандидатов (id + title + description + tags), возвращает `{ "matched_ids": [...] }`, `max_tokens=220`.

## Вызовы LLM на один запрос

| Шаг | Вызов | Условие |
|-----|-------|---------|
| 1 | `analyze_dialog_scenario` (routing) | Всегда |
| 2 | `extract_movie_params` | Если intent = movie_selection |
| 3 | `_ai_rank_films_by_query` (ранжирование) | Если movie_selection + нет keyword-match |
| 4 | `call_llm` (генерация ответа) | Всегда кроме lead_* и UI-команд |

**Итого:** обычно 1-2 вызова, максимум 4 вызова на один запрос пользователя.

## Формат контекста

Контекст строится как конкатенация разделов через `\n\n`. Каждый раздел начинается с заголовка `=== Название раздела ===`.

### Пример контекста для `movie_selection`

```
[содержимое docs/work/10_company_knowledge.md]

=== Подобранные фильмы ===
• Космическое путешествие | Возраст: 7+ | Длительность: 15 минут | Захватывающее...
• Солнечная система | Возраст: 7+ | Длительность: 12 минут | ...
```

### Пример контекста для `franchise_info`

```
[содержимое docs/work/10_company_knowledge.md]

=== Частые вопросы (FAQ) ===
[Общие вопросы]
В: Что такое VR-кинотеатр?
О: ...

=== Информация о франшизе ===
[Краткое описание франшизы]
...
[Условия и инвестиции]
...
```

## Формат сессионной памяти

**Таблица:** `ai_sessions`  
**Поля:** `telegram_user_id`, `active_intent`, `state_json`, `updated_at`, `expires_at`

### Структура `state_json`

```json
{
  "_active_intent": "movie_selection",
  "params": {
    "theme": "космос",
    "grade": 3,
    "duration": "d15",
    "raw_query": "космос для 3 класса до 15 минут",
    "last_recommended_ids": [12, 47, 88]
  },
  "ai_current_item_id": 47,
  "ai_current_item_title": "Космическое путешествие",
  "ai_flow_step": "showing_results",
  "history": [
    {"role": "user", "content": "фильм про космос для 3 класса"},
    {"role": "assistant", "content": "Вот 3 варианта..."}
  ]
}
```

**Лимит истории:** 8 сообщений (4 обмена).  
**TTL:** 30 минут без активности (настраивается через `AI_SESSION_TTL_MINUTES`).

## База знаний (`docs/work/10_company_knowledge.md`)

> **[СЛУЖЕБНЫЙ ФАЙЛ]** Этот файл читается AI напрямую через `ai_context.load_company_knowledge()` и передаётся в системный промпт. Редактировать с осторожностью — изменения немедленно влияют на ответы бота.

**Кеш:** файл загружается один раз в `_knowledge_cache` (in-memory). Для обновления без перезапуска нужно добавить инвалидацию кеша или перезапустить бот.

## Как менять поведение AI

| Задача | Файл | Что изменить |
|--------|------|-------------|
| Изменить тон / правила ответа | `ai_answer.py` | `_BASE_RULES` |
| Добавить/изменить инструкцию для intent | `ai_answer.py` | `_INTENT_INSTRUCTIONS[intent]` |
| Добавить новый intent | `ai_router.py`, `ai_decision.py`, `ai_answer.py` | Добавить scoring-функцию, обновить списки, добавить инструкцию |
| Добавить тему для подбора фильмов | `ai_catalog.py` | `_THEME_KEYWORDS` |
| Изменить маппинг класс → возраст | `ai_catalog.py` | `_GRADE_TO_AGE` |
| Обновить базу знаний о компании | `docs/work/10_company_knowledge.md` | Редактировать файл + перезапустить бот |
| Сменить модель | `.env` | `AI_MODEL=...` |
| Увеличить лимит ответа | `.env` | `AI_MAX_TOKENS=...` |
