## Фаза 7 — Админский раздел

### Реализованные возможности

**Доступ**
- Команда `/admin`
- Проверка по `ADMIN_TELEGRAM_ID` на каждом handler
- Для неадмина: «Доступ запрещён»

**Заявки**
- Список с сортировкой: new → in_progress → done (внутри — новые сверху)
- Пагинация 10/страница
- Карточка с полными данными
- Смена статуса: «В работу» / «Закрыть»

**Вопросы**
- Список (неотвеченные сначала, потом отвеченные)
- Пагинация 10/страница
- Карточка с полным текстом и историей ответа
- Ответ пользователю через FSM (AdminStates.waiting_reply)
- «Отметить отвеченным» без отправки ответа
- Если сообщение пользователю не доставлено — статус не меняется

**Статистика**
- Периоды: Сегодня / 7 дней / 30 дней / Всё время
- Метрики: пользователи, заявки (по типам), вопросы, аналитические события

**Аналитика**
- Таблица `analytics_events`
- Логируются: open_main_menu, open_catalog, open_catalog_item, start_lead_form, submit_lead, ask_question

## Следующие приоритеты

| Задача | Сложность | Ценность |
|--------|----------|---------|
| Webhook-режим вместо polling | Средняя | Снижение latency |
| Redis для FSM и кеша | Средняя | Масштабируемость |
| Кнопка «Открыть на сайте» с трекингом | Низкая | Статистика переходов |
| Автоматический запуск парсера по расписанию | Средняя | Актуальность каталога |
| Шифрование персональных данных | Средняя | Compliance |
| Экспорт заявок в CSV | Низкая | Операционное удобство |

## Ограничения первой версии

- Один администратор (ADMIN_TELEGRAM_ID)
- MemoryStorage для FSM (сбрасывается при перезапуске)
- `click_site_link` не заполняется (нет кнопки с трекингом переходов)
- Нет уведомлений администратору о новых заявках в реальном времени (только при загрузке /admin)
