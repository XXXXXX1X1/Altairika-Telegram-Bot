# 01 — Продуктовая рамка

## 1. Контекст и цель проекта

Altairika — бренд с сайтом altairika.ru, предлагающий:
- каталог услуг / контента для частных клиентов (точный формат — см. ниже),
- франшизу для предпринимателей.

**Проблема:** потенциальный клиент или партнёр попадает на сайт, не всегда понимает, с чего начать, не имеет быстрого канала для консультации и часто уходит без действия.

**Цель бота:** сократить путь от интереса до целевого действия (запись на сеанс / заявка на франшизу / оставленный контакт) через удобный диалог в Telegram.

---

## 2. Тип контента каталога

> [Требует данных] Точный тип сущностей в каталоге (фильмы / оздоровительные программы / процедуры / другое) необходимо уточнить, изучив сайт altairika.ru. В документации используется нейтральный термин **«карточка»** для единицы каталога и **«сеанс»** для целевого действия. Если каталог содержит фильмы — терминология обновится соответственно (сеанс = показ, карточка = карточка фильма, изображение = постер).

---

## 3. Целевая аудитория

### Персона A — «Потенциальный клиент»

- Хочет узнать, что предлагает Altairika, и записаться на сеанс.
- Задаёт вопросы о конкретных позициях каталога: что это, для кого, как проходит, сколько стоит.
- Ключевое действие: **запись на сеанс или звонок**.

### Персона B — «Потенциальный франчайзи»

- Рассматривает открытие своего объекта под брендом Altairika.
- Интересуется условиями, инвестициями, поддержкой, конкурентным контекстом.
- Ключевое действие: **заявка на переговоры / оставленный контакт**.

### Персона C — «Исследователь»

- Сравнивает Altairika с конкурентами перед принятием решения.
- Может совпадать с Персоной A или B на разных этапах.
- Ключевое действие: **получить убедительный ответ и перейти к целевому действию**.

> [Решение] Три персоны не изолированы: один пользователь может последовательно пройти через сценарии A → C → A. Бот не требует выбора роли в начале — он адаптируется к вопросам.

---

## 4. Ключевые функции бота

### Обязательные [Ядро]

1. **Каталог** — фильтрация по категориям, просмотр карточек с возможностью записи.
2. **Карточка** — полное описание позиции каталога, изображение (если есть), кнопка записи / оставления контакта.
3. **Запись / контакт** — форма сбора имени и номера телефона, с подтверждением.
4. **Франшиза** — структурированный раздел с ответами на ключевые вопросы, форма заявки.
5. **FAQ** — фиксированные ответы на частые вопросы, сгруппированные по темам.

### Желательные [Улучшение]

6. **Сравнение с конкурентами** — динамически, по открытым данным.
7. **Напоминание** — если пользователь не завершил оформление, вернуть через N часов.
8. **История просмотров** — что пользователь смотрел в текущей сессии.
9. **Похожие позиции** — в конце карточки, по категории.
10. **Административная панель** — обновление каталога без деплоя.

---

## 5. Ограничения и нефункциональные требования

- Бот работает только в Telegram (мобильный и десктоп).
- Ответ на любое действие пользователя — не более 3 секунд.
- Навигация не должна засорять чат: изменение текущего сообщения предпочтительнее отправки нового.
- Данные каталога должны обновляться без перезапуска бота (через парсинг или API).
- Сбор контактов соответствует требованиям согласия на обработку персональных данных.

## 6. Метрики успеха

| Метрика | Ориентир | Примечание |
|---------|----------|------------|
| Конверсия в контакт | > 15% от старта | [Гипотеза] требует уточнения после запуска |
| Конверсия в заявку на франшизу | > 5% от зашедших в раздел | [Гипотеза] |
| Доля завершённых форм | > 70% от начатых | |
| Среднее время до первого действия | < 2 мин | |
| Процент отказов (покинули без действия) | < 50% | |

> [Гипотеза] Все ориентиры — стартовые гипотезы. Реальные бенчмарки определяются после первых 2 недель работы.

---

## 7. Что НЕ входит в скоуп

- Онлайн-оплата сеансов (принимается только заявка / контакт).
- Интеграция с CRM на старте (контакты сохраняются в базе бота и передаются вручную или экспортируются).
- Мобильное приложение.
- Веб-чат (только Telegram).
- Автоматическое расписание (запись передаётся оператору вручную или через имеющуюся CRM).
