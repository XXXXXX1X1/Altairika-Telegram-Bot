# 03 — Интерфейс и UX

## 1. Принципы навигации

### Изменение vs. новое сообщение

Telegram позволяет редактировать существующее сообщение через `editMessageText` / `editMessageCaption`. Это предотвращает засорение чата и создаёт ощущение «экрана» вместо «переписки».

**Правило: редактировать текущее сообщение, если:**
- переход происходит по inline-кнопке в рамках одного раздела,
- содержимое — только текст или текст + кнопки (без смены медиа).

**Правило: отправлять новое сообщение, если:**
- открывается карточка с изображением (нельзя превратить текстовое сообщение в медиа),
- начинается диалоговая форма (шаг за шагом — каждый вопрос = отдельное сообщение),
- отправляется финальное подтверждение или уведомление.

> [Решение] Карточки с изображением всегда открываются в новом сообщении. Все остальные переходы — редактирование. Список карточек — редактирование.

## 2. Главное меню

```
┌─────────────────────────────────────────────────────┐
│  🎬  Добро пожаловать в Altairika!                  │
│                                                     │
│  Выберите, что вас интересует:                      │
└─────────────────────────────────────────────────────┘
┌────────────────────┐  ┌────────────────────┐
│   📋  Каталог      │  │   🏢  Франшиза     │
└────────────────────┘  └────────────────────┘
┌────────────────────┐  ┌────────────────────┐
│   ❓  Вопрос       │  │   📞  Контакт      │
```

> [Требует данных] Иконки уточняются после определения типа контента (фильмы, процедуры, программы). Текст приветствия — после получения голоса бренда от заказчика.

**Вызов главного меню:** `/start`, `/menu`, или кнопка «Главное меню» в любом экране.

## 3. Каталог

### 3.1 Экран категорий (фильтры)

```
┌─────────────────────────────────────────────────────┐
│  📋 Каталог Altairika                               │
│                                                     │
│  Выберите категорию или просмотрите всё:            │
└─────────────────────────────────────────────────────┘
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  Все (N)     │  │  Категория 1 │  │  Категория 2 │
└──────────────┘  └──────────────┘  └──────────────┘
         [← Главное меню]
```

> [Требует данных] Категории и количество позиций (N) формируются при парсинге каталога. В документации не фиксируются до получения реальных данных.

> [Решение] Если категорий больше 6 — переход на вертикальный список (по одной кнопке в строку). Если одна категория — экран категорий пропускается, сразу список карточек.

### 3.2 Список карточек

```
┌─────────────────────────────────────────────────────┐
│  📋 Категория: [Название]  (показано 1–5 из 12)     │
│                                                     │
│  1. Название позиции A                              │
│  2. Название позиции B                              │
│  3. Название позиции C                              │
│  4. Название позиции D                              │
│  5. Название позиции E                              │
└─────────────────────────────────────────────────────┘
┌──────┐  ┌────────────────┐  ┌──────┐
│  ←   │  │   1 / 3        │  │  →   │
└──────┘  └────────────────┘  └──────┘
┌──────────────────┐  ┌──────────────────┐
│  Открыть [1]     │  │  Открыть [2]     │
└──────────────────┘  └──────────────────┘
... (кнопки для каждой позиции на странице)
[← Назад к категориям]
```

> [Решение] По 5 позиций на страницу. Пагинация — встроенная (редактируется текущее сообщение). Кнопки открытия карточек — под списком, пронумерованы.

### 3.3 Карточка позиции — без изображения

```
┌─────────────────────────────────────────────────────┐
│  Название позиции                                   │
│                                                     │
│  Описание: 2–5 строк текста о том, что это,        │
│  для кого, как проходит.                            │
│                                                     │
│  Категория: [X]  │  Длительность: [Y]              │
│  Цена: [Z] руб.  (если есть в данных)              │
└─────────────────────────────────────────────────────┘
┌─────────────────────┐  ┌─────────────────────┐
│   ✅ Записаться      │  │   ← Назад к списку  │
└─────────────────────┘  └─────────────────────┘
┌─────────────────────┐  ┌─────────────────────┐
│   💬 Задать вопрос  │  │   🔗 Похожие        │
└─────────────────────┘  └─────────────────────┘
```

### 3.4 Карточка позиции — с изображением

```
[Изображение / постер отправляется как photo]

Caption:
───────────────────────────────────────
Название позиции

Описание: 2–5 строк.

Категория: [X] | Длительность: [Y]
Цена: [Z] руб.
───────────────────────────────────────

┌─────────────────────┐  ┌─────────────────────┐
│   ✅ Записаться      │  │   ← Назад к списку  │
└─────────────────────┘  └─────────────────────┘
┌─────────────────────┐  ┌─────────────────────┐
│   💬 Задать вопрос  │  │   🔗 Похожие        │
└─────────────────────┘  └─────────────────────┘
```

> [Решение] Caption в Telegram ограничен 1024 символами. Если описание длиннее — оно усекается с «...» и предлагается «Читать подробнее» (открывает новое текстовое сообщение).

> [Решение] Кнопка «Назад к списку» в карточке с изображением не редактирует текущее (фото) сообщение, а отправляет новое текстовое с восстановленным списком. Изображение остаётся в истории — это нормально.

## 4. Форма записи и сбора контакта

Форма запускается из карточки или из главного меню («Контакт»).

```
ШАГ 1
───────────────────────────────────────
  📝 Как вас зовут?
  
  Напишите ваше имя:

ШАГ 2
───────────────────────────────────────
  📞 Ваш номер телефона?
  
  [📱 Поделиться контактом]
  или напишите номер вручную:

ШАГ 3 (только для записи на конкретную позицию)
───────────────────────────────────────
  📅 Удобное время?
  (опционально — если не указать, менеджер уточнит)
  
  [Пропустить]
  или напишите пожелание:

ШАГ 4 — Подтверждение
───────────────────────────────────────
  Проверьте данные:
  
  Имя: [Имя]
  Телефон: [Номер]
  Позиция: [Название] (если есть)
  Время: [Пожелание или «не указано»]
  
  [✅ Всё верно, отправить]  [✏️ Изменить]

ШАГ 5 — Завершение
───────────────────────────────────────
  ✅ Заявка принята!
  
  Мы свяжемся с вами в ближайшее рабочее время.
  
  [← Вернуться в каталог]  [Главное меню]
```

> [Решение] Шаг с временем — опциональный. Не все пользователи знают удобное время заранее; его отсутствие не должно блокировать отправку заявки.

> [Решение] Кнопка «Поделиться контактом» в Telegram передаёт верифицированный номер. Ввод вручную — резервный вариант, так как не все пользователи захотят делиться.

> [Гипотеза] Шаг с временем можно убрать, если отдел продаж предпочитает уточнять время при звонке. Проверить с заказчиком.

## 6. FAQ

Структурированный раздел с ответами на типичные вопросы.

```
[Список тем]
    ┌────────────────────────────────┐
    │  О сеансах и записи            │
    │  О франшизе                    │
    │  Цены и оплата                 │
    │  Другой вопрос                 │
    └────────────────────────────────┘

→ [Список вопросов темы]  →  [Ответ]  →  [← Назад к вопросам]
                                           [📞 Задать свой вопрос]
```

**Свободный текст вне формы:**

Если пользователь пишет произвольный текст (не в форме), бот отвечает:

```
«Я работаю через кнопки. Возможно, вам поможет раздел «Частые вопросы»,
или оставьте вопрос — менеджер ответит в рабочее время.»

[❓ Частые вопросы]  [✉️ Написать вопрос]
```

> [Решение] Разбор намерений свободного текста не реализуется. Простое, предсказуемое поведение лучше, чем ненадёжный keyword-matching.

## 7. Сравнение с конкурентами

```
┌─────────────────────────────────────────────────────┐
│  📊 Altairika vs. рынок                             │
│                                                     │
│  Параметр          | Altairika | Конк. 1 | Конк. 2 │
│  ──────────────────|──────────|─────────|─────────│
│  [Параметр A]      │    ✅     │   ⚠️    │   ❌    │
│  [Параметр B]      │    ✅     │   ✅    │   ⚠️    │
│  [Параметр C]      │    ✅     │   ❌    │   ❌    │
│  ...               │          │         │         │
└─────────────────────────────────────────────────────┘
  *Данные по открытым источникам, актуальны на [дату]

[← Назад]  [📞 Записаться]  [❓ Задать вопрос]
```

> [Требует данных] Список конкурентов и параметры сравнения определяются при исследовании рынка. В код ничего не зашивается — хранится в базе данных.

> [Гипотеза] Таблица в Telegram читается плохо на узких экранах. Альтернатива — структурированный список («По параметру X: Altairika — ..., конкуренты — ...»). Решение принять после тестирования на реальных устройствах.

## 8. Обработка ошибок и граничные состояния

| Ситуация | Реакция бота |
|----------|-------------|
| Пользователь ввёл не номер телефона | «Похоже, это не номер телефона. Попробуйте ещё раз или нажмите кнопку "Поделиться".» |
| Пользователь прервал форму | «Продолжить оформление или выйти?» + кнопки |
| Бот не понял вопрос | Стандартный ответ + предложение связаться |
| Каталог пуст / ошибка загрузки | «Каталог временно недоступен. Оставьте контакт — мы ответим.» |
| Длинный ответ LLM | Разбивается на несколько сообщений или усекается с «...» |

## 9. Тон и голос бота

> [Требует данных] Тон определяется брендингом Altairika. До получения гайдлайнов — умолчания:

- Вежливый, не формальный.
- Обращение на «вы».
- Никаких восклицательных знаков в каждом предложении.
- Конкретный: вместо «Мы рады вам помочь!» — «Выберите раздел или задайте вопрос.»
- Не использует слова «потрясающий», «уникальный», «лучший» без фактических обоснований.
