# Altairika Telegram Bot

Telegram-бот для компании Altairika — выездного VR-кинотеатра для школ и мероприятий. Бот помогает клиентам изучить каталог VR-фильмов, оформить заявку, задать вопрос, узнать об условиях франшизы. Для сотрудников предусмотрена встроенная панель администратора.

## Возможности

**Для пользователей**
- Каталог VR-фильмов с фильтрами по возрасту, длительности и предметам
- Карточки фильмов с постером, описанием и кнопкой перехода на сайт
- Заявки: бронирование показа, запрос по франшизе, контакт
- Раздел FAQ — вопросы и ответы из базы данных
- Раздел «Франшиза» — питч, условия, поддержка, преимущества, FAQ
- Сравнение с конкурентами
- Свободный текст — предложение FAQ или вопрос оператору

**Для администратора** (команда `/admin`)
- Список заявок с сортировкой по статусу и дате, пагинация
- Карточка заявки, смена статуса (новая → в работе → закрыта)
- Список вопросов пользователей, фильтр непрочитанных
- Ответ пользователю прямо из панели (FSM-форма, редактирование на месте)
- Статистика: пользователи, заявки, вопросы, события — за сегодня / 7 дней / 30 дней / всё время
- Уведомления о новых заявках и вопросах с кнопкой быстрого перехода

## Быстрый старт (Docker)

### 1. Клонировать репозиторий

```bash
git clone https://github.com/XXXXXX1X1/Altairika-Telegram-Bot.git
cd Altairika-Telegram-Bot
```

### 2. Создать файл `.env`

```bash
cp .env.example .env
```

Открыть `.env` в любом редакторе и заполнить три строки:

```env
BOT_TOKEN=1234567890:AABBCCDDEEFFaabbccddeeff   # Токен от @BotFather
DATABASE_URL=postgresql+asyncpg://bot:secret@localhost:5432/altairika
ADMIN_TELEGRAM_ID=123456789                       # Ваш Telegram user ID
```

> Узнать свой `ADMIN_TELEGRAM_ID` можно через [@userinfobot](https://t.me/userinfobot).  
> `DATABASE_URL` в этом файле используется только для локального запуска; внутри Docker он подставляется автоматически.

### 3. Запустить

```bash
docker compose up --build -d
```

**Что происходит автоматически при каждом запуске:**

| Шаг | Что делается |
|-----|-------------|
| 1 | Поднимается PostgreSQL 16, ожидается готовность базы |
| 2 | Применяются все миграции Alembic (`alembic upgrade head`) |
| 3 | Заполняется FAQ (статические данные) |
| 4 | Заполняется раздел «Сравнение с конкурентами» |
| 5 | Синхронизируется контент франшизы с `altairika.ru` (если сайт доступен) |
| 6 | Синхронизируется каталог фильмов с `altairika.ru` (если сайт доступен) |
| 7 | Запускается бот |

Шаги 5 и 6 обращаются к внешнему сайту. Если сайт недоступен — бот всё равно запустится, а соответствующие разделы покажут заглушку до следующего перезапуска.

### 4. Проверить логи

```bash
docker compose logs -f bot
```

Ожидаемый вывод при успешном старте:

```
=====================================================
  Altairika Bot — подготовка к запуску
=====================================================

[1/5] Применяем миграции базы данных...
      Миграции применены.

[2/5] Заполняем FAQ...
[3/5] Заполняем сравнение с конкурентами...
[4/5] Синхронизируем контент франшизы с altairika.ru...
[5/5] Синхронизируем каталог фильмов с altairika.ru...

=====================================================
  Запускаем бота...
=====================================================

INFO:aiogram.event:Bot started (polling)
```

### 5. Остановить

```bash
docker compose down          # остановить контейнеры
docker compose down -v       # + удалить volume с данными БД
```

## Локальный запуск (без Docker)

### Требования

- Python 3.11+
- PostgreSQL 16 (запущен и доступен)

### Установка

```bash
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Настройка `.env`

```env
BOT_TOKEN=...
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/altairika
ADMIN_TELEGRAM_ID=...
```

### Первый запуск (выполнить один раз)

```bash
alembic upgrade head           # Создать таблицы
python scripts/seed_faq.py     # FAQ
python scripts/seed_compare.py # Сравнение с конкурентами
python scripts/seed_franchise.py  # Контент франшизы (нужен интернет)
python scripts/run_parser.py   # Каталог фильмов (нужен интернет)
```

### Запуск бота

```bash
python -m bot
```

## Панель администратора

Доступ открывается командой `/admin`. Бот проверяет `ADMIN_TELEGRAM_ID` на каждом запросе — все остальные пользователи получают отказ.

**Разделы панели:**

| Раздел | Описание |
|--------|----------|
| Заявки | Список всех/новых заявок, карточка, смена статуса |
| Вопросы | Список всех/неотвеченных вопросов, FSM-форма ответа |
| Статистика | 13 метрик за 4 периода: сегодня / 7 дней / 30 дней / всё время |

Ответ пользователю отправляется прямо из Telegram — форма редактируется на месте, не создавая новых сообщений.

## Обновление данных

Синхронизация контента с сайта запускается автоматически при каждом перезапуске бота. Для принудительной синхронизации без перезапуска:

```bash
# Каталог фильмов
docker compose exec bot python scripts/run_parser.py

# Контент франшизы
docker compose exec bot python scripts/seed_franchise.py
```

## Миграции

```bash
# Применить все миграции
alembic upgrade head

# Откатить последнюю миграцию
alembic downgrade -1

# Создать новую миграцию
alembic revision --autogenerate -m "описание"
```

Файлы миграций: `alembic/versions/`
- `0001_initial.py` — все базовые таблицы
- `0002_admin.py` — поля ответов на вопросы и таблица `analytics_events`

## Разработка

### Порядок регистрации роутеров

Порядок `dp.include_router()` в `__main__.py` имеет значение:
1. `start_router` — команда `/start`
2. `admin_router` — команда `/admin` и панель
3. `lead_router`, `faq_router` — FSM-формы (приоритет перед остальными)
4. `catalog_router`, `franchise_router`, `compare_router`
5. `freetext_router` — последним, catch-all для свободного текста

### Навигация

Вся навигация реализована через редактирование существующего сообщения (`edit_message_text` / `edit_message_caption`). Новое сообщение создаётся только в двух случаях: ответ на команду `/start` и карточка с фото при переходе из текстового экрана.
